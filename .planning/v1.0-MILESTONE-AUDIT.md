## Tech Debt by Phase

### Phase 01: Infrastructure + DevOps (4 items)

**1. health_stations replay safety (LOW)**
`ON CONFLICT DO NOTHING` in `0002_seed_gis_data.py` has no conflict target column for `health_stations`. Replaying migration after a downgrade+upgrade cycle will double-insert all 32 stations. Fix: add `UNIQUE (psgc_code)` constraint to `health_stations` in a future migration and change to `ON CONFLICT (psgc_code) DO NOTHING`.
File: `backend/alembic/versions/0002_seed_gis_data.py` line ~103

**2. celery_beat has no healthcheck (LOW)**
`docker-compose.yml` has a healthcheck for `celery_worker` (`celery inspect ping`) but not for `celery_beat`. Beat is unused in Phase 1 (no tasks registered), but Phase 7 (scheduled ML jobs) will depend on it being alive.
File: `docker-compose.yml` celery_beat service

**3. nginx proxy not covered by automated tests (INFO)**
`test_health.py` sends requests to `http://localhost:8000/health` (direct uvicorn port). The nginx reverse proxy path (`http://localhost/api/health`) was validated in UAT but has no pytest coverage. An nginx configuration regression would not be caught by `pytest tests/test_infra/`.
File: `backend/tests/test_infra/test_health.py`

**4. audit_logs.record_id has no DB default (INFO)**
`audit_logs.record_id UUID NOT NULL` — no `gen_random_uuid()` default is set in migration 0001. Phase 2's audit writer must supply the UUID explicitly on every INSERT. This is workable but unexpected for a field that looks auto-generated.
File: `backend/alembic/versions/0001_initial_schema.py`

**Total: 4 items, 0 blockers**
