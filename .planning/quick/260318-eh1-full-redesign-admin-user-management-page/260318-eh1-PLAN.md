<objective>
Redesign the admin user management page to replace the side-panel (Sheet) create flow with a dedicated full-page create user experience at /admin/users/new. The form must be modular: split into independent section components (Identity, Credentials, Roles, BHS Assignment) so future fields can be added without touching the page layout.

Purpose: Side panels are cramped and interrupt the user's spatial context. A dedicated page gives the form room to breathe, allows progressive disclosure of sections, and is easier to extend. Field nurses and admins using tablets need large touch targets and clear visual hierarchy.

Output: CreateUserPage.tsx (dedicated route), UserFormSections.tsx (modular section components), updated UsersPage.tsx (link-based navigation), updated App.tsx (new route registration).
</objective>

Full-page layout. Owner of all form state. Imports section components from `@/features/admin/components/UserFormSections`.

Layout structure:
```
<div className="min-h-screen bg-background">
  <div className="max-w-2xl mx-auto px-6 py-8 space-y-8">
    {/* Page header with back navigation */}
    <header>
      <Button variant="ghost" size="sm" onClick={() => navigate("/admin/users")} ...>
        <ArrowLeft .../> Back to Users
      </Button>
      <h1 className="text-2xl font-semibold text-foreground mt-4">Create User</h1>

        Provision a new staff account for the system.
      </p>
    </header>

    {/* Form sections — vertically stacked with dividers */}
    <div className="space-y-8 divide-y divide-border">
      <IdentitySection ... />
      <div className="pt-8"><CredentialsSection ... /></div>
      <div className="pt-8"><RolesSection ... /></div>
      <div className="pt-8"><BhsAssignmentSection ... /></div>
    </div>

    {/* Sticky footer action bar — stays visible while scrolling on tablet */}
    <div className="sticky bottom-0 bg-background border-t border-border pt-4 pb-6 flex gap-3 justify-end">
      <Button variant="outline" onClick={() => navigate("/admin/users")} disabled={isLoading}>
        Cancel
      </Button>
      <Button onClick={handleSubmit} disabled={isLoading || !canSubmit} className="bg-primary hover:bg-primary/90 min-w-[120px]">
        {isLoading ? <spinner /> : "Create User"}
      </Button>
    </div>
  </div>
</div>
```

State the page owns:
- `fullName`, `email`, `password`, `showPassword`, `selectedRoles`, `healthStationId`
- `isLoading`, `emailError`, `roleError`

`canSubmit` = `fullName.trim().length > 0 && email.trim().length > 0 && password.length > 0 && selectedRoles.length > 0`

`handleSubmit`:
1. Set `isLoading = true`, clear errors
2. Call `createUser({ email, full_name: fullName, password, roles: selectedRoles, health_station_id: isSystemAdmin ? null : healthStationId })`
3. On success: `toast.success("User created successfully.")` then `navigate("/admin/users")`
4. On 409: `setEmailError("This email is already in use.")`
5. On 422 with "system_admin" in detail: `setRoleError("system_admin cannot be combined with other roles.")`
6. On other error: `toast.error("Something went wrong. Try again.")`
7. Finally: `setIsLoading(false)`

Use `useNavigate` from react-router-dom.
