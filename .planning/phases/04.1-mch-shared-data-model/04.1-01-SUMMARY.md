# Phase 4.1 Plan 01: MCH ORM Models Summary

**One-liner:** Eight MCH SQLAlchemy ORM models (prenatal/postpartum/EPI/nutrition enrollments and visits) with offline sync fields, ML trigger flags, and BHS isolation via enrollment-only health_station_id.

## What Was Built

All 8 MCH ORM model classes across 4 new model files, following the established Consultation model pattern from Phase 3.

### Files Created

**backend/app/models/prenatal.py**
- `PrenatalEnrollment`: enrollment table with patient/BHS/user FKs, gravida/para/LMP/EDC clinical fields, `is_high_risk` ML flag, `next_visit_date` for overdue detection, offline sync fields
- `PrenatalVisit`: full prenatal vitals (aog_weeks, weight, BP, fundic_height, FHT, hgb, tt_dose, iron supplementation), NO health_station_id

**backend/app/models/postpartum.py**
- `PostpartumEnrollment`: nullable `prenatal_enrollment_id` FK (ON DELETE SET NULL), `day1_date`/`week1_date`/`week6_date` postpartum schedule columns, delivery_type, birth_outcome
- `PostpartumVisit`: visit_type (day1|week1|week6), wound_status, breastfeeding, family_planning_counseling, newborn_status, NO health_station_id

**backend/app/models/epi.py**
- `EpiEnrollment`: enrollment_date, `fic_status` ML flag, `next_visit_date` for earliest upcoming dose
- `EpiVaccination`: TEXT vaccine column (not ENUM), dose_number, date_given, lot_number, site, administered_by, unique constraint on (enrollment, vaccine, dose_number)

**backend/app/models/nutrition.py**
- `NutritionEnrollment`: enrollment_date, no next_visit_date (monthly weighing sessions)
- `NutritionVisit`: weight/height/muac, WHO Z-scores (waz/haz/whz), nutrition_status, `severe_wasting` ML flag (WHZ < -3)

**backend/app/models/__init__.py** (modified)
- Added imports for all 8 new models
- `__all__` extended from 6 to 14 entries

## Architecture Decisions Made

| Decision | Rationale |
|----------|-----------|
| Visit tables have no `health_station_id` | BHS isolation via JOIN through enrollment avoids denormalization across 4 extra tables |
| `epi_vaccinations.vaccine` is TEXT not ENUM | DOH may add vaccines; TEXT + Pydantic validation avoids ALTER TYPE migration churn |
| `PostpartumEnrollment.prenatal_enrollment_id` nullable | External facility deliveries have no matching prenatal record in the system |
| `NutritionEnrollment` has no `next_visit_date` | OPT+ weighing driven by monthly sessions, not per-patient scheduling |
| UniqueConstraint on (enrollment_id, vaccine, dose_number) | Prevents duplicate dose recording at the DB level |
