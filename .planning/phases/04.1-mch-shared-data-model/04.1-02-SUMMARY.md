## Decisions Made

- `create_type=False` pattern for the `record_status` ENUM variable passed into `op.create_table` — required because `postgresql.ENUM` fires `CREATE TYPE` for each table it appears in; calling `.create(checkfirst=True)` explicitly first and then using `create_type=False` prevents duplicate creation errors in SQLAlchemy + asyncpg
- EpiVaccinationCreate uses `vaccine: EPI_VACCINES` (Pydantic Literal) not a PostgreSQL ENUM column — matches ORM model's `Text` column type; DOH vaccine schedule updates will only require a Pydantic change, not a migration
- NutritionVisitCreate deliberately excludes `waz`, `haz`, `whz`, `nutrition_status`, `severe_wasting` — these are computed by service layer on save; accepting them from clients would allow spoofing ML-derived health classifications

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Fixed ENUM double-creation error in Alembic migration**
- **Found during:** Task 2 (first upgrade attempt)
- **Issue:** `postgresql.ENUM(create_type=True)` was set on both the explicit `.create()` call and the variable passed to `op.create_table`. SQLAlchemy fires `CREATE TYPE` before each table that uses the ENUM, causing `DuplicateObjectError: type "record_status" already exists` on the second table.
- **Fix:** Created two ENUM variables — one with `create_type=True` for the explicit `.create(checkfirst=True)` call, and one with `create_type=False` for use inside all `op.create_table` calls. The transactional DDL rolled back the first failed attempt cleanly.
- **Files modified:** `backend/alembic/versions/0005_mch_schema.py`
- **Verification:** `alembic upgrade head` succeeds; `alembic downgrade 0004` then `alembic upgrade head` both succeed on re-run
- **Committed in:** `62fa916` (Task 2 commit)

## Issues Encountered

- SQLAlchemy `postgresql.ENUM` fires `CREATE TYPE` automatically via a `before_create` hook for each table column that references it — even after an explicit `.create(checkfirst=True)` call. The `create_type=False` parameter suppresses this per-table auto-creation hook. Fixed inline per deviation Rule 1.
