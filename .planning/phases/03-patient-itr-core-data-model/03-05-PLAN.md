**Create `frontend/src/pages/patients/ConsultationSheet.tsx`:**

Build the consultation form as a Sheet side panel (per CONTEXT.md locked decision: "Add Consultation: Button on the profile page — opens a Sheet side panel with consultation form"):

Props:
```typescript
interface ConsultationSheetProps {
  open: boolean;
  onOpenChange: (open: boolean) => void;
  patientId: number;
  patientName: string;       // for confirmation strip
  healthStationName: string; // for confirmation strip
  onSuccess: () => void;     // callback to refresh consultations table
}
```

1. **Sheet structure:**
   - Use shadcn `Sheet`, `SheetContent`, `SheetHeader`, `SheetTitle`, `SheetDescription` composition
   - `SheetContent` opens from the right side (default), `className="sm:max-w-[540px] overflow-y-auto"`
   - `SheetTitle`: "New Consultation"
   - `SheetDescription`: Patient identity confirmation — patient name + BHS as Badge, compact read-only display

2. **Form sections using FieldSet + FieldLegend:**

   **Section 1 - Chief Complaint (FieldSet):**
   - `Textarea` (required), full width, minimum 3 rows, autofocus
   - Label: "Chief Complaint *"
   - `aria-required="true"`

   **Section 2 - Vitals (FieldSet):**
   - Background: `var(--card)` with `var(--border)` border, rounded-lg padding
   - Single column layout in Sheet (Sheet is narrow), two-column on wider sheets if space permits
   - All fields optional, `inputmode="numeric"` attribute on all number inputs
   - Input height: minimum 48px (`min-h-[48px]`) — gloved hands, mobile tap targets
   - Font size in inputs: 16px minimum (prevents iOS zoom)
   - Unit labels: right-aligned inside InputGroup (use FieldGroup from field.tsx)

   Fields:
   - BP: Paired inputs `[Systolic] / [Diastolic]` with visible "/" separator, unit "mmHg"
     - Systolic: `Input type="number"`, step 1, inputmode="numeric"
     - Diastolic: `Input type="number"`, step 1, inputmode="numeric"
   - Heart Rate: `Input type="number"`, unit "bpm"
   - Respiratory Rate: `Input type="number"`, unit "breaths/min"
   - Temperature: `Input type="number"`, step 0.1, unit "°C"
   - Weight: `Input type="number"`, step 0.1, unit "kg"
   - Height: `Input type="number"`, step 0.1, unit "cm"
   - **BMI (computed display):** Read-only text field. Update live when both weight and height are filled:
     ```
     bmi = weight / ((height / 100) ** 2)
     ```
     Display rounded to 1 decimal. Show Tooltip on hover: "Computed from weight and height". If weight or height missing, show "--".

   **Section 3 - Additional Vitals (Collapsible, collapsed by default):**
   - Use shadcn `Collapsible` component
   - Trigger label: "Show additional vitals" / "Hide additional vitals"
   - O2 Saturation: `Input type="number"`, unit "%"
   - Blood Glucose: `Input type="number"`, unit "mg/dL"
   - These values go into `vitals_extra` object in the API request

   **Section 4 - Assessment (FieldSet):**
   - Diagnosis: `Textarea` optional, minimum 2 rows
   - Label: "Diagnosis (ICD-10 or description)"

   **Section 5 - Referral (FieldSet):**
   - Referring To: `Input type="text"` optional
   - Label: "Refer to (if applicable)"

3. **Submit flow:**
   - Call `createConsultation(patientId, body)` on submit
   - Build `vitals_extra` from additional vitals section (o2_saturation, blood_glucose) — only include non-null values
   - Success: toast "Consultation saved", call `onSuccess()` callback to refresh table, then call `onOpenChange(false)` to close Sheet
   - Error: toast "Could not save consultation. Check your connection and try again."
   - Submit button: "Save Consultation" (primary, full-width). Disabled during async with Spinner.

4. **Cancel:** "Cancel" secondary Button closes Sheet via `onOpenChange(false)`

5. **Validation:**
   - chief_complaint is required — show FieldError "Chief complaint is required." if empty on submit
   - Vitals range validation is handled by backend Pydantic schemas; frontend relies on `type="number"` min/max attributes matching backend ranges (bp_systolic min=40 max=300, bp_diastolic min=20 max=200, heart_rate min=20 max=300, respiratory_rate min=4 max=80, temperature min=30.0 max=45.0, weight min=0.5 max=500, height min=20 max=300)

6. **Form reset:** When Sheet opens (open changes to true), reset all form fields to defaults so previous entries don't persist.
