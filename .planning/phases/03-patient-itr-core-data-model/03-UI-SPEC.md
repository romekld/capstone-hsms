## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline badge padding, field helper text offset |
| sm | 8px | Compact element spacing, input internal padding supplement |
| md | 16px | Default field spacing, card internal padding, button padding |
| lg | 24px | Section header-to-content gap, form section padding |
| xl | 32px | Layout column gap, page section breaks |
| 2xl | 48px | Major page section separation |
| 3xl | 64px | Page-level vertical rhythm (top/bottom page padding) |

Exceptions:
- Touch targets on all interactive controls: minimum 44√ó44px (ui-ux-pro-max `touch-target-size` rule ‚Äî field nurses use tablets, may use gloves)
- Vitals input grid cells: minimum 48px height to allow deliberate tapping of individual fields
- Row actions in tables (kebab menu / row click): minimum 44px row height

## Color

All values via CSS variables from `globals.css` ‚Äî never use raw OKLCH/hex in components.

| Role | CSS Variable | OKLCH (light) | Usage |
|------|-------------|---------------|-------|
| Dominant (60%) | `hsl(var(--background))` ‚Üí `var(--background)` | oklch(1 0 0) | Page background, main content area |
| Secondary (30%) | `var(--card)` / `var(--sidebar)` | oklch(1 0 0) / oklch(0.985 0 0) | Patient list cards, profile header card, sidebar, table rows |
| Accent (10%) | `var(--primary)` | oklch(0.205 0 0) | Reserved ‚Äî see accent list below |
| Destructive | `var(--destructive)` | oklch(0.577 0.245 27.325) | Destructive actions only |
| Status: Critical | `var(--status-critical)` | oklch(53% 0.22 27) | `possible_duplicate` badge, alert variant for duplicate warning |
| Status: Safe | `var(--status-safe)` | oklch(58% 0.16 145) | Active patient badge, successful save indicator |
| Status: Warning | `var(--status-warning)` | oklch(68% 0.17 68) | Read-only city-wide access indicator badge |
| BHS tier | `var(--bhs-tier)` | oklch(55% 0.14 240) | BHS name badge on city-wide search results |
| Muted text | `var(--muted-foreground)` | oklch(0.556 0 0) | Helper text, empty state body, column sub-labels |

Accent (`var(--primary)`) reserved for:
1. Primary action buttons ("Register Patient", "Save Consultation", "Save Changes")
2. Active navigation item in sidebar
3. Focus ring on form inputs (via `var(--ring)`)
4. "Use existing patient" action link in duplicate warning card
5. Sortable column header sort indicator arrow

Never apply accent color to decorative elements, card borders, or secondary labels.

## Surface & Layout Contracts

### Route: `/patients` ‚Äî Patient Search Page

- Page h1 (Display/28px): "Patients"
- Top bar: search `Input` (full width on mobile, 400px max on desktop) + "Register Patient" `Button` (primary, right-aligned)
- Search scope toggle: `Checkbox` + label "Search all BHS" ‚Äî appears inline with search bar; hidden for CHO/DSO/coordinator (always city-wide for them)
- Results: `Table` with columns: Patient Name | Birthdate | Sex | BHS | Registered ‚Äî sortable on all columns
- BHS column: `Badge` using `var(--bhs-tier)` for city-wide results; suppressed when scope is own-BHS
- Pagination: shadcn `Pagination` component, 20 rows per page default
- Row click: navigates to `/patients/:id`
- Loading state: `Skeleton` rows (5 placeholder rows, matching table column widths)
- Empty state: see Copywriting Contract

### Route: `/patients/new` ‚Äî Register Patient (Dedicated Page)

- This is a dedicated page, NOT a sheet or modal. Source: user clarification decision.
- Page h1 (Display/28px): "Register Patient"
- Back navigation: Breadcrumb `Patients > Register Patient` using shadcn `Breadcrumb`
- Form layout: single-column on mobile; two-column grid on tablet+ (768px+)
- Form sections (use `FieldSet` + `FieldLegend` from existing `field.tsx`):
  1. **Identity** ‚Äî Last Name (required), First Name (required), Middle Name (optional)
  2. **Demographics** ‚Äî Birthdate (required, `Input type="date"`), Sex (required, `Select`: Male / Female)
  3. **Address** ‚Äî Barangay (required, `Select` from barangay list), Address Line (optional, free text)
  4. **Contact** ‚Äî Mobile Number (optional, `Input type="tel"`)
  5. **Assignment** ‚Äî Health Station (required, `Select` ‚Äî pre-filled to nurse's own BHS, editable by admin)
- Required field indicator: asterisk (*) next to `FieldLabel`, with a legend note "* Required" above the submit button
- Vitals: NOT collected at registration ‚Äî vitals belong to consultation records only
- Submit button: "Register Patient" (primary, full-width on mobile, auto-width on desktop) ‚Äî disabled during async operation (ui-ux-pro-max `loading-buttons` rule)
- Cancel: secondary `Button` "Cancel" navigates back to `/patients`

### Duplicate Warning State (on `/patients/new`)

- Triggered after submit when backend returns 409 or duplicate match
- Display as `Alert` component (variant: destructive using `var(--status-critical)`) inline above form submit area ‚Äî NOT a blocking modal
- Warning card content: existing patient's full name, BHS, registration date, link to their record
- Two action buttons inside the warning card:
  1. "Use Existing Patient" ‚Äî `Button` variant outline ‚Äî navigates to existing patient's profile; cancels current registration
  2. "Register Anyway" ‚Äî `Button` variant ghost with `var(--status-critical)` text color ‚Äî triggers audit-logged save with `possible_duplicate = true`
- "Register Anyway" must not be the visually dominant action ‚Äî it is subordinate to "Use Existing Patient"

### Route: `/patients/:id` ‚Äî Patient Profile Page

- Page h1 (Display/28px): Patient's full name
- Sub-header: Breadcrumb `Patients > [Patient Name]`
- **Profile header card** (`Card` component): Displays name, birthdate, sex, computed age, BHS ‚Äî all read-only. No mobile/address in header (CONTEXT.md decision).
  - `Badge` for sex: neutral styling
  - `Badge` for `possible_duplicate = true`: `var(--status-critical)` background, label "Possible Duplicate"
  - Read-only city-wide access indicator: `Badge` with `var(--status-warning)` background, label "View Only" ‚Äî shown when nurse is viewing another BHS's patient
- **Consultations tab**: Only tab in Phase 3. No placeholder tabs for future programs (CONTEXT.md decision ‚Äî clean, functional, no empty sections).
- Add Consultation button: placed at top-right of the Consultations tab panel, label "Add Consultation" ‚Äî primary `Button`. Opens a **dedicated page** (see below). NOT a sheet panel. Source: user clarification decision.
- Consultations table: columns ‚Äî Date | Chief Complaint | Diagnosis | Referring To | Recorded By ‚Äî default sort newest first, sortable on Date column
- Empty state for consultations: see Copywriting Contract

### Route: `/patients/:id/consultations/new` ‚Äî Add Consultation (Dedicated Page)

- This is a dedicated page, NOT a sheet or modal. Source: user clarification decision.
- Page h1 (Display/28px): "New Consultation"
- Sub-header: Breadcrumb `Patients > [Patient Name] > New Consultation`
- Patient identity confirmation strip (read-only): Name + BHS shown as compact summary row above the form ‚Äî so nurse can confirm they are on the right patient
- Form sections (use `FieldSet` + `FieldLegend`):
  1. **Chief Complaint** ‚Äî `Textarea` (required), full width, minimum 3 rows; label "Chief Complaint *"
  2. **Vitals** ‚Äî two-column grid on tablet+ (768px+), single-column on mobile
     - BP Systolic / BP Diastolic: paired inputs with slash separator, both `Input type="number"`, unit label "mmHg"
     - Heart Rate: `Input type="number"`, unit label "bpm"
     - Respiratory Rate: `Input type="number"`, unit label "breaths/min"
     - Temperature: `Input type="number"` step="0.1", unit label "¬∞C"
     - Weight: `Input type="number"` step="0.1", unit label "kg"
     - Height: `Input type="number"` step="0.1", unit label "cm"
     - BMI: computed display field (read-only, updates live when weight/height are filled), not an input
     - All vitals optional; backend validates range only when provided
  3. **Additional Vitals** (collapsible `Collapsible` ‚Äî collapsed by default):
     - O2 Saturation: `Input type="number"`, unit label "%"
     - Blood Glucose: `Input type="number"`, unit label "mg/dL"
     - Trigger label: "Show additional vitals" / "Hide additional vitals"
  4. **Assessment** ‚Äî Diagnosis: `Textarea` optional, minimum 2 rows; label "Diagnosis (ICD-10 or description)"
  5. **Referral** ‚Äî Referring To: `Input type="text"` optional; label "Refer to (if applicable)"
- Submit button: "Save Consultation" (primary, full-width on mobile) ‚Äî disabled during async
- Cancel: secondary `Button` "Cancel" navigates back to `/patients/:id`

### Vitals Input ‚Äî Field Worker Ergonomics

These rules apply to all vitals inputs in the consultation form:
- Input height: minimum 48px (not the default 36px) ‚Äî gloved hands, mobile tap targets
- Font size in inputs: 16px minimum ‚Äî prevents iOS auto-zoom on focus
- Number inputs: `inputmode="numeric"` attribute ‚Äî surfaces numeric keyboard on mobile
- Unit labels: rendered as right-aligned inline text inside the input group (use `FieldGroup` from `field.tsx`) ‚Äî not as placeholder text
- BP inputs: rendered as a pair `[Systolic] / [Diastolic]` with a visible "/" separator between ‚Äî standard clinical reading format
- Vitals section background: slightly elevated surface to visually group it from other form sections ‚Äî use `var(--card)` background with `var(--border)` border, `--radius-lg` corners

## Accessibility Requirements

From ui-ux-pro-max skill, Priority 1 (CRITICAL) and Priority 2 (CRITICAL):

- All form inputs have associated `Label` (via `Field` / `FieldLabel` components from `field.tsx`)
- All icon-only buttons (row kebab, sort arrows) have `aria-label` attributes
- Color is never the sole indicator ‚Äî duplicate warning uses text + icon + color
- Focus rings visible on all interactive elements (`ring-2 ring-ring ring-offset-2`)
- Tab order matches visual DOM order on all forms
- `aria-required="true"` on all required inputs
- `aria-invalid="true"` + `aria-describedby` pointing to `FieldError` when field has a validation error
- Minimum contrast ratio 4.5:1 for all body text on its background
- `prefers-reduced-motion` respected ‚Äî no forced entrance animations on form pages
- `inputmode="numeric"` on all number inputs (vitals, BP fields)
- Keyboard navigation: Enter submits focused form; Escape on breadcrumb back focus returns to table row last focused
