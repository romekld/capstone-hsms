# Plan 03-01: Data Model + Migration

## Objective
Create the `patients` and `consultations` SQLAlchemy ORM models and a single Alembic migration `0004_patients_and_consultations.py` covering both tables, all indexes, and the PostgreSQL `sex_enum` ENUM type.

## Context
- Phase: 03 - Patient ITR + Core Data Model
- Depends on: Phase 2 complete (`health_stations`, `barangays`, `users` tables exist)
- Blocked by: INFRA-04 known bug — verify GIS data mount in `docker-compose.yml` is correct before running `alembic upgrade head` (migration chain runs 0001→0002→0003→0004; 0002 seeds GIS data)

## Tasks

### Task 1: Create Patient ORM model

**Files to create/modify:**
- `backend/app/models/patient.py` — new Patient SQLAlchemy model
- `backend/app/models/__init__.py` — add Patient + Consultation imports

**Implementation notes:**
- Use `class Patient(TimestampMixin, SoftDeleteMixin, Base)` — both mixins are mandatory per CLAUDE.md and `backend/app/core/base.py`; `SoftDeleteMixin` wires the `do_orm_execute` hook that auto-filters `deleted_at IS NULL` on every SELECT
- Column spec (all from CONTEXT.md §Patient Identity Fields):
  - `id: Mapped[int]` — `Integer, primary_key=True, autoincrement=True`
  - `last_name: Mapped[str]` — `Text, nullable=False`
  - `first_name: Mapped[str]` — `Text, nullable=False`
  - `middle_name: Mapped[str | None]` — `Text, nullable=True`
  - `birthdate: Mapped[date]` — `Date, nullable=False`
  - `sex: Mapped[str]` — `PG_ENUM("male", "female", name="sex_enum"), nullable=False`; import as `from sqlalchemy.dialects.postgresql import ENUM as PG_ENUM`
  - `barangay_psgc_code: Mapped[str]` — `Text, ForeignKey("barangays.psgc_code"), nullable=False`
  - `address_line: Mapped[str | None]` — `Text, nullable=True`
  - `health_station_id: Mapped[int]` — `Integer, ForeignKey("health_stations.id"), nullable=False, index=True`
  - `mobile_number: Mapped[str | None]` — `Text, nullable=True`
  - `possible_duplicate: Mapped[bool]` — `Boolean, nullable=False, default=False, server_default=sa.false()`
- Relationships — ALL must use `lazy="raise"` (mandatory per CLAUDE.md):
  - `health_station = relationship("HealthStation", lazy="raise")`
  - `barangay = relationship("Barangay", lazy="raise")`
  - `consultations = relationship("Consultation", back_populates="patient", lazy="raise")`
- Do NOT add `local_id UUID` or `status record_status` — explicitly deferred to Phase 9
- Do NOT store BMI as a column — it is computed in the Pydantic schema
- After writing `patient.py`, update `backend/app/models/__init__.py` to add:
  ```python
  from app.models.patient import Patient  # noqa: F401
  from app.models.consultation import Consultation  # noqa: F401
  ```
  and extend `__all__` accordingly. This is required so Alembic `autogenerate` detects the models.

### Task 3: Create Alembic migration `0004_patients_and_consultations.py`

**Files to create/modify:**
- `backend/alembic/versions/0004_patients_and_consultations.py` — single migration covering both tables

**Implementation notes:**
- File location: `backend/alembic/versions/0004_patients_and_consultations.py`
- Set `revision = "0004"` and `down_revision = "0003"` to chain after the users migration
- `upgrade()` must execute in this exact order (order matters — FK dependencies and ENUM creation):
  1. `op.execute("CREATE TYPE sex_enum AS ENUM('male', 'female')")` — MUST come before `op.create_table("patients", ...)`, otherwise `ProgrammingError: type "sex_enum" does not exist`
  2. `op.create_table("patients", ...)` with all columns listed in Task 1. Use `sa.Column("sex", postgresql.ENUM("male", "female", name="sex_enum", create_type=False), nullable=False)` — `create_type=False` prevents Alembic from trying to re-create the ENUM it cannot find
  3. `op.execute("CREATE INDEX ix_patients_duplicate_check ON patients (lower(last_name), lower(first_name), birthdate)")` — B-tree index for duplicate detection; must use `op.execute()` with raw DDL because functional indexes cannot be expressed via `op.create_index()` alone
  4. `op.execute("CREATE INDEX ix_patients_name_search ON patients USING GIN (to_tsvector('simple', last_name || ' ' || first_name))")` — GIN index for full-text name search; MUST use `op.execute()` with raw DDL — `op.create_index()` cannot express tsvector computed expressions
  5. `op.create_table("consultations", ...)` with all columns from Task 2, including `sa.ForeignKey("patients.id")` on `patient_id`
  6. `op.create_index("ix_consultations_patient_id", "consultations", ["patient_id"])` — B-tree index for fast patient history queries; this one CAN use `op.create_index()` since it is a plain column index
- `downgrade()` must reverse in the opposite order:
  1. `op.drop_index("ix_consultations_patient_id", table_name="consultations")`
  2. `op.drop_table("consultations")`
  3. `op.execute("DROP INDEX IF EXISTS ix_patients_name_search")`
  4. `op.execute("DROP INDEX IF EXISTS ix_patients_duplicate_check")`
  5. `op.drop_table("patients")`
  6. `op.execute("DROP TYPE IF EXISTS sex_enum")`
- Import block at top: `import sqlalchemy as sa; from alembic import op; from sqlalchemy.dialects import postgresql`
- Do NOT use `op.execute()` for the plain B-tree index on `consultations.patient_id` — use `op.create_index()` for readability
