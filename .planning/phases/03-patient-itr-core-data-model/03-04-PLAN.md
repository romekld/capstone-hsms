# Plan 03-04: Patients Feature Module — Types, API Client, and Route Registration

## Objective
Create the `features/patients/` TypeScript module (types, API functions) and register all four patient routes in `App.tsx` with the correct `ProtectedRoute` role guard, plus add the Patients nav item to the sidebar.

## Context
- Phase: 03 - Patient ITR + Core Data Model
- Depends on: 03-02 and 03-03 (backend endpoints must exist for API functions to target correct URLs)
- Blocked by: Nothing — this plan creates the shared foundation that Plans 03-05, 03-06, and 03-07 consume

## Tasks

### Task 1: Create `features/patients/types.ts` and `features/patients/api.ts`

**Files to create/modify:**
- `frontend/src/features/patients/types.ts` — TypeScript interfaces mirroring backend Pydantic schemas
- `frontend/src/features/patients/api.ts` — async API functions using the existing axios instance

**Implementation notes for `types.ts`:**
- Mirror `backend/app/schemas/patient.py` exactly. All field names must match the JSON keys returned by the API.
- `Patient` interface:
  ```typescript
  export interface Patient {
    id: number;
    display_id: string;         // computed: "BHS03-20260001"
    last_name: string;
    first_name: string;
    middle_name: string | null;
    birthdate: string;          // ISO 8601 date string "YYYY-MM-DD"
    sex: "male" | "female";
    barangay_psgc_code: string;
    address_line: string | null;
    health_station_id: number;
    bhs_name: string | null;    // from joinedload in service
    mobile_number: string | null;
    possible_duplicate: boolean;
    read_only: boolean;         // derived by service — true when BHS-level role views another BHS's patient
    created_at: string;         // ISO 8601 datetime
    updated_at: string;
  }
  ```
- `PatientCreate` interface:
  ```typescript
  export interface PatientCreate {
    last_name: string;
  ```

- `PatientSearchParams` interface:

**Implementation notes for `api.ts`:**
- Import `api` from `"@/lib/axios"` — the existing axios instance with auth interceptor and token refresh queue
- Functions (from RESEARCH.md §Pattern 8):
  ```typescript
  export async function listPatients(params?: PatientSearchParams): Promise<Patient[]>
  // GET /patients with params as query string

  export async function getPatient(patientId: number): Promise<Patient>
  // GET /patients/{patientId}

  export async function createPatient(body: PatientCreate): Promise<Patient>
  // POST /patients — 201; throws on 409 (duplicate found) or 4xx/5xx

  export async function checkDuplicate(
    last_name: string,
    first_name: string,
    birthdate: string,
  ): Promise<DuplicateMatch[]>
  // GET /patients/duplicate-check?last_name=...&first_name=...&birthdate=...

  export async function listConsultations(patientId: number): Promise<Consultation[]>
  // GET /patients/{patientId}/consultations

  export async function createConsultation(
    patientId: number,
    body: ConsultationCreate,
  ): Promise<Consultation>
  // POST /patients/{patientId}/consultations — 201
  ```
- The `createPatient` function may receive a 409 response when duplicate found. The frontend page (03-05) handles 409 by checking `error.response?.status === 409` and reading `error.response?.data?.detail?.duplicates`. Do not swallow the error in `api.ts` — let it propagate.

### Task 2: Register patient routes in `App.tsx` and update sidebar

**Files to create/modify:**
- `frontend/src/App.tsx` — add four patient routes under a new `ProtectedRoute` block
- `frontend/src/layouts/app-sidebar.tsx` (or wherever `NAV_ITEMS` is defined) — add Patients nav item

**Implementation notes for `App.tsx`:**
- Add imports at the top of `App.tsx` (follow the existing import pattern for admin pages — lazy imports if admin pages use lazy loading, otherwise named imports):
  ```typescript
  import { PatientsPage } from "@/pages/patients/PatientsPage";
  import { RegisterPatientPage } from "@/pages/patients/RegisterPatientPage";
  import { PatientProfilePage } from "@/pages/patients/PatientProfilePage";
  import { AddConsultationPage } from "@/pages/patients/AddConsultationPage";
  ```
  Note: these page files do not exist yet (created in 03-05, 03-06, 03-07). Add the imports now — TypeScript will error until the page files are created, but the route config is correct.
- Add a new `ProtectedRoute` block AFTER the admin block and BEFORE the default redirects:
  ```tsx
  {/* Protected — clinical staff (all roles except system_admin and bhw) */}
  <Route
    element={
      <ProtectedRoute
        allowedRoles={[
          "nurse",
          "midwife",
          "physician",
          "city_health_officer",
          "disease_surveillance_officer",
          "phis_coordinator",
        ]}
      />
    }
  >
    <Route element={<AppShell />}>
      <Route path="/patients" element={<PatientsPage />} />
      <Route path="/patients/new" element={<RegisterPatientPage />} />
      <Route path="/patients/:id" element={<PatientProfilePage />} />
      <Route path="/patients/:id/consultations/new" element={<AddConsultationPage />} />
    </Route>
  </Route>
  ```
- BHW is intentionally excluded — zero BHW access in Phase 3 (CONTEXT.md confirmed)
