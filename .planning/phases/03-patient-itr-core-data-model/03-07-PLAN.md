# Plan 03-07: Patient Profile Page and Add Consultation Page

## Objective
Implement `/patients/:id` (profile header card + Consultations tab with sortable table) and `/patients/:id/consultations/new` (dedicated consultation form with discrete vitals, collapsible additional vitals, live BMI, and proper ergonomics for tablet field nurses).

## Context
- Phase: 03 - Patient ITR + Core Data Model
- Depends on: 03-04 (types + API), 03-05 (PatientFormSections for shared field patterns)
- Blocked by: Nothing beyond 03-04 and 03-05

## Tasks

### Task 1: Build the Patient Profile Page

**Files to create/modify:**
- `frontend/src/pages/patients/PatientProfilePage.tsx` — the `/patients/:id` route component

**Implementation notes:**

**Page structure (from UI-SPEC §Route: `/patients/:id`):**
- Page entrance: `className="login-page-enter"` on outermost container (reuse globals.css animation)
- Fetch patient on mount: `getPatient(Number(params.id))` — 404 → `toast.error(...)` + `navigate("/patients")`; 403 → `navigate("/unauthorized")`
- Breadcrumb: `Patients > [Patient Full Name]` — the second crumb is not a link (current page)
- Page h1: Patient's full name — `{patient.last_name}, {patient.first_name} {patient.middle_name ?? ""}` — Display/28px (`text-2xl font-semibold`). This is the focal point per UI-SPEC.

**Profile header card (UI-SPEC §Route: `/patients/:id` — Header card fields: name, birthdate, sex, age, BHS):**
- Use shadcn `Card` with `className="bg-card border border-border rounded-[--radius-lg] p-6"`
- Fields displayed read-only (no inputs):
  - Full name (same as h1, can be omitted from card if h1 already shows it — or show in the card as primary identity confirmation)
  - Birthdate: formatted as human-readable (e.g., `"January 1, 1990"`)
  - Computed age: `Math.floor((today - birthdate) / 365.25)` in years — label "Age", value e.g. `"36 years"`
  - Sex: `Badge` neutral styling (`capitalize(patient.sex)` → "Male" or "Female")
  - BHS: `Badge` using `style={{ color: "var(--bhs-tier)" }}` — `patient.bhs_name`
- Conditional badges in the header card:
  - `patient.possible_duplicate === true`: `Badge` with `style={{ backgroundColor: "var(--status-critical)", color: "white" }}`, label `"Possible Duplicate"` (UI-SPEC: `var(--status-critical)` background)
  - `patient.read_only === true`: `Badge` with `style={{ backgroundColor: "var(--status-warning)", color: "white" }}`, label `"View Only"` (UI-SPEC: `var(--status-warning)`)
  - The "View Only" badge confirms to nurses that this is another BHS's patient; they can view but not add consultations
- No mobile_number or address in the header card — confirmed in CONTEXT.md decision

**Consultations tab (the ONLY tab — no placeholder tabs):**
- Use shadcn `Tabs` with a single `TabsTrigger` labeled `"Consultations"` and a single `TabsContent`
- No placeholder tabs for Phase 4–8 programs — confirmed in CONTEXT.md and UI-SPEC
- "Add Consultation" `Button` variant `"default"` (primary) at top-right of the `TabsContent` header: navigates to `/patients/${patient.id}/consultations/new`. Hidden (not rendered) when `patient.read_only === true` — nurses cannot add consultations to another BHS's patient
- Consultations table columns: Date | Chief Complaint | Diagnosis | Referring To | Recorded By (UI-SPEC §Route: `/patients/:id`)
- Date column: formatted as `"MMM D, YYYY h:mm a"` — newest first (default sort by `created_at` desc per CONTEXT.md)
- Date column is sortable (toggle asc/desc): manage `sortDir` state, client-side sort the fetched array
- Row click: no action in Phase 3 (no consultation detail page yet — that is Phase 4+). Rows are not `cursor-pointer`
- Empty state (no consultations): centered with heading `"No consultations recorded"` and body `"Add the first consultation record for this patient."` (exact copy from UI-SPEC §Copywriting)
- Loading state: 3 `Skeleton` rows while `consultationsLoading` is true
- Error state: `toast.error("Could not load consultations.")` in catch block

**State management:**
```typescript
const { id } = useParams<{ id: string }>();
const [patient, setPatient] = useState<Patient | null>(null);
const [consultations, setConsultations] = useState<Consultation[]>([]);
const [patientLoading, setPatientLoading] = useState(true);
const [consultationsLoading, setConsultationsLoading] = useState(true);
const [sortDir, setSortDir] = useState<"asc" | "desc">("desc");  // newest first
```

Fetch both in parallel on mount:
```typescript
useEffect(() => {
  const patientId = Number(id);
  Promise.all([
    getPatient(patientId),
    listConsultations(patientId),
  ]).then(([p, c]) => {
    setPatient(p);
    setConsultations(c);
  }).catch(...).finally(() => {
    setPatientLoading(false);
    setConsultationsLoading(false);
  });
}, [id]);
```

### Task 2: Build the Add Consultation Page

**Files to create/modify:**
- `frontend/src/pages/patients/AddConsultationPage.tsx` — the `/patients/:id/consultations/new` route component

**Implementation notes:**

**Page structure (from UI-SPEC §Route: `/patients/:id/consultations/new`):**
- Page entrance animation: `className="login-page-enter"`
- Container: `max-w-2xl mx-auto px-6 py-8 space-y-8`
- Breadcrumb: `Patients > [Patient Name] > New Consultation`. Fetch patient name on mount via `getPatient(patientId)` to populate the breadcrumb dynamically.
- Page h1: `"New Consultation"` — Display/28px
- Patient identity confirmation strip (read-only, above the form): shows `"{patient.last_name}, {patient.first_name}"` and `"{patient.bhs_name}"` as a compact row — confirms the nurse is on the correct patient (UI-SPEC: "so nurse can confirm they are on the right patient")

**Form sections (from UI-SPEC §Route: `/patients/:id/consultations/new`):**

1. **Chief Complaint** (`FieldSet` + `FieldLegend`):
   - `Textarea` required, `autoFocus`, min 3 rows
   - Label: `"Chief Complaint *"` with `aria-required="true"`
   - Inline error if empty on submit: `"Chief Complaint is required."`

2. **Vitals** (two-column grid on tablet+, single column on mobile — `md:grid md:grid-cols-2 md:gap-4`):
   - Section background: `var(--card)` background with `var(--border)` border, `--radius-lg` corners (UI-SPEC §Vitals Input — "slightly elevated surface")
   - All vitals inputs: `Input type="number"`, min height 48px (`className="h-12"`), `inputmode="numeric"`, font size 16px minimum (UI-SPEC §Vitals Input)
   - Use `FieldGroup` from `field.tsx` to display unit labels as right-aligned inline text (NOT placeholder) — e.g., `"mmHg"`, `"bpm"`, `"°C"`, `"kg"`, `"cm"`
   - BP: paired `[Systolic Input] / [Diastolic Input]` with visible `"/"` separator (UI-SPEC: "rendered as a pair with a visible '/' separator")
   - Fields: BP Systolic + BP Diastolic (paired), Heart Rate, Respiratory Rate, Temperature (step="0.1"), Weight (step="0.1"), Height (step="0.1")
   - BMI: read-only computed display field — update live when weight or height changes:
     ```typescript
     const bmi = weight && height && height > 0
       ? (weight / Math.pow(height / 100, 2)).toFixed(1)
       : null;
     ```
     Wrap in `Tooltip` with content `"Computed from weight and height"` (UI-SPEC §Component Inventory). Display as `<span>{bmi ?? "—"}</span>`

3. **Additional Vitals** (shadcn `Collapsible` — collapsed by default):
   - Trigger label: `"Show additional vitals"` (collapsed) / `"Hide additional vitals"` (expanded) — exact copy from UI-SPEC
   - Inside: O2 Saturation (`Input type="number"` step="1", unit `"%"`), Blood Glucose (`Input type="number"` step="0.1", unit `"mg/dL"`)
   - Both map to `vitals_extra: { o2_saturation: number, blood_glucose: number }` in `ConsultationCreate`
   - `Collapsible` open/close: default Base UI Collapsible animation 200ms (UI-SPEC §Animation Contract)

4. **Assessment** (`FieldSet` + `FieldLegend` labeled `"Assessment"`):
   - Diagnosis `Textarea` optional, min 2 rows, label `"Diagnosis (ICD-10 or description)"`

5. **Referral** (`FieldSet` + `FieldLegend` labeled `"Referral"`):
   - Referring To `Input type="text"` optional, label `"Refer to (if applicable)"`

**Submit flow:**
1. Validate `chief_complaint` non-empty; stop and show inline error if empty
2. `setIsLoading(true)` — disables submit button
3. Build `ConsultationCreate` payload — include only non-empty vitals (omit undefined/null values to avoid backend 422 from null numeric fields)
4. Call `createConsultation(patientId, payload)`
5. On success: `toast.success("Consultation saved.")`, `navigate("/patients/" + patientId)`
6. On error: `toast.error("Could not save consultation. Check your connection and try again.")` (exact copy from UI-SPEC)

**Sticky footer action bar:**
- `"Back to [Patient Name]"` `Button` variant `"secondary"` — navigates to `/patients/${patientId}` (exact copy from UI-SPEC §Copywriting)
- `"Save Consultation"` `Button` variant `"default"` primary — `disabled={isLoading}` with loading spinner
- Full-width on mobile, auto-width on desktop

**Accessibility (from UI-SPEC §Accessibility):**
- All number inputs have `inputmode="numeric"` (numeric keyboard on mobile)
- `autoFocus` on Chief Complaint `Textarea` — focal point per UI-SPEC
- `aria-required="true"` on Chief Complaint
- `prefers-reduced-motion` respected — wrap all `transition` and `animate-*` classes in a `useReducedMotion()` check or use `motion-safe:` Tailwind prefix
