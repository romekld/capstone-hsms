# Plan 03-02: Patient Backend тАФ Schemas + Repository + Service + Router

## Objective
Deliver the complete patient backend: Pydantic schemas, `PatientRepository` (BHS-scoped and city-wide search, duplicate check), `PatientService` (business logic, audit logging, read-only enforcement), and the FastAPI patients router with RBAC.

## Context
- Phase: 03 - Patient ITR + Core Data Model
- Depends on: 03-01 (Patient + Consultation ORM models and migration must exist)
- Blocked by: Nothing beyond 03-01

## Tasks

### Task 1: Create Pydantic schemas for patients

**Files to create/modify:**
- `backend/app/schemas/patient.py` тАФ PatientCreate, PatientResponse, PatientSearchParams, DuplicateMatch

**Implementation notes:**
- All schemas use `model_config = ConfigDict(from_attributes=True)` (Pydantic v2 pattern, same as `backend/app/schemas/user.py`)
- `PatientCreate` тАФ request body for POST /patients:
  - Required: `last_name: str`, `first_name: str`, `birthdate: date`, `sex: Literal["male", "female"]`, `barangay_psgc_code: str`, `health_station_id: int`
  - Optional: `middle_name: str | None = None`, `address_line: str | None = None`, `mobile_number: str | None = None`
  - Include `force_create: bool = False` тАФ when True, service skips the duplicate check guard and sets `possible_duplicate=True` on the saved record
- `PatientResponse` тАФ response for GET /patients and POST /patients:
  - All Patient model fields: `id`, `last_name`, `first_name`, `middle_name`, `birthdate`, `sex`, `barangay_psgc_code`, `address_line`, `health_station_id`, `mobile_number`, `possible_duplicate`, `created_at`, `updated_at`
  - Computed field: `display_id: str` тАФ formatted as `BHS{health_station_id:02d}-{created_at.year}{id:04d}` (e.g., `BHS03-20260001`). Use Pydantic v2 `@computed_field` тАФ NOT stored in the database
  - `read_only: bool = False` тАФ set True by service layer when BHS-scoped role views another BHS's patient; defaults False. This is not a model column тАФ set after construction in service layer
  - `bhs_name: str | None = None` тАФ health station name for display; populated by service via joinedload
- `PatientSearchParams` тАФ query parameters for GET /patients:
  - `q: str = ""` тАФ search term (name or display_id prefix)
  - `city_wide: bool = False` тАФ toggle; always True for CROSS_BHS_ROLES (enforced in service)
  - `limit: int = 20`
  - `offset: int = 0`
- `DuplicateMatch` тАФ returned when duplicate(s) found:
  - `id: int`, `last_name: str`, `first_name: str`, `middle_name: str | None`, `birthdate: date`, `health_station_id: int`, `bhs_name: str | None`, `created_at: datetime`, `display_id: str`
  - Inherit `@computed_field display_id` from PatientResponse or duplicate the property

### Task 2: Create PatientRepository

**Files to create/modify:**
- `backend/app/repositories/patient.py` тАФ PatientRepository(BaseRepository)

**Implementation notes:**
- `class PatientRepository(BaseRepository)` тАФ inherits from `backend/app/repositories/base.py`; `self.session` and `self.user` are available from `__init__`
- All methods are `async def` тАФ never use sync SQLAlchemy
- Use `from sqlalchemy import select, func` and `from sqlalchemy.orm import joinedload`
- **`get_by_id(patient_id: int) -> Patient | None`**: `select(Patient).where(Patient.id == patient_id).options(joinedload(Patient.health_station))`. Apply `_isolation_filter(stmt, Patient)` so BHS-scoped roles cannot GET another BHS's patient by ID. Return `result.scalar_one_or_none()`
- **`search_bhs_scoped(q: str, limit: int, offset: int) -> list[Patient]`**: Full-text search within own BHS. Build `stmt = select(Patient).options(joinedload(Patient.health_station))`. If `q` is non-empty, add `.where(func.to_tsvector("simple", Patient.last_name + " " + Patient.first_name).op("@@")(func.to_tsquery("simple", q.strip() + ":*")))`. Apply `_isolation_filter(stmt, Patient)`. Add `.limit(limit).offset(offset).order_by(Patient.last_name, Patient.first_name)`. Note: use `"simple"` config тАФ NOT `"english"` тАФ because names must not be stemmed (RESEARCH.md ┬зPattern 1)
- **`search_city_wide(q: str, limit: int, offset: int) -> list[Patient]`**: Same query as above but WITHOUT `_isolation_filter()`. Per CONTEXT.md and RESEARCH.md ┬зPattern 3, city-wide search is a separate method тАФ never add a `bypass` flag to `_isolation_filter()`. If `q` is empty, return all patients ordered by `last_name`
- **`find_duplicates(last_name: str, first_name: str, birthdate: date) -> list[Patient]`**: City-wide exact case-insensitive match. `select(Patient).options(joinedload(Patient.health_station)).where(func.lower(Patient.last_name) == last_name.lower(), func.lower(Patient.first_name) == first_name.lower(), Patient.birthdate == birthdate)`. Do NOT call `_isolation_filter()` тАФ duplicate check is always city-wide (CONTEXT.md decision). The `do_orm_execute` SoftDeleteMixin hook still applies (only active records are checked); this is an accepted limitation documented in RESEARCH.md ┬зPitfall 5
- **`create(data: dict) -> Patient`**: `patient = Patient(**data)`, `self.session.add(patient)`, `await self.session.flush()`, `await self.session.refresh(patient)`, return `patient`. Do not commit тАФ service layer manages transactions
- **`count_bhs_scoped(q: str) -> int`** and **`count_city_wide(q: str) -> int`**: Count variants of search methods for pagination; use `select(func.count()).select_from(Patient)` with the same WHERE clause

### Task 3: Create PatientService + patients router

**Files to create/modify:**
- `backend/app/services/patient.py` тАФ PatientService
- `backend/app/routers/patients.py` тАФ FastAPI patients router
- `backend/app/routers/__init__.py` тАФ register patients router
- `backend/app/main.py` (or equivalent app factory) тАФ include patients router

**Implementation notes:**

**PatientService (`backend/app/services/patient.py`):**
- `class PatientService`: `__init__(self, session: AsyncSession, current_user: UserSchema)` тАФ instantiates `PatientRepository(session=session, user=current_user)`
- Import `CROSS_BHS_ROLES` from `backend/app/repositories/base.py` to derive read-only flag
- **`async def search(params: PatientSearchParams) -> list[PatientResponse]`**: Determine if user is cross-BHS role (`is_cross_bhs = any(r in CROSS_BHS_ROLES for r in self.current_user.roles)`). If `is_cross_bhs` or `params.city_wide`, call `repo.search_city_wide()`; else call `repo.search_bhs_scoped()`. For each patient, set `read_only=True` if user is NOT cross-BHS and `patient.health_station_id != self.current_user.health_station_id`
- **`async def get_patient(patient_id: int) -> PatientResponse`**: Call `repo.get_by_id(patient_id)`. Raise `HTTPException(404)` if not found. Raise `HTTPException(403)` if BHS-scoped user requests a patient outside their BHS (service-layer guard, belt-and-suspenders on top of repo isolation)
- **`async def check_duplicates(last_name: str, first_name: str, birthdate: date) -> list[DuplicateMatch]`**: Call `repo.find_duplicates(...)`. Map results to `DuplicateMatch` schema
- **`async def create_patient(data: PatientCreate) -> PatientResponse`**: If `data.force_create` is False, run `check_duplicates()`; if results found, raise `HTTPException(409, detail={"duplicates": [...]})` тАФ the frontend handles 409 by showing the warning card. If `data.force_create` is True (nurse clicked "Register Anyway"): set `possible_duplicate=True` in the save dict; after flush, write audit log entry (see Pattern 7 in RESEARCH.md); commit
- **Audit log write for duplicate override**: Use raw SQL `INSERT INTO audit_logs` via `session.execute(text(...))` with `gen_random_uuid()` for `record_id`. Store `performed_by_user_id` in `new_values` JSONB (not in the `performed_by` column which is NULL per RESEARCH.md ┬зPattern 7). Commit AFTER both the patient insert and audit insert тАФ they must be atomic

**Patients router (`backend/app/routers/patients.py`):**
- Router prefix: `/patients`, tags: `["patients"]`
- Do NOT use router-level `dependencies=[require_role(...)]` тАФ routes have different role requirements:
  - `GET /patients` тАФ nurse, midwife, physician, city_health_officer, disease_surveillance_officer, phis_coordinator
  - `GET /patients/duplicate-check` тАФ nurse, midwife (only registering roles need this)
  - `GET /patients/{patient_id}` тАФ same as GET /patients
  - `POST /patients` тАФ nurse, midwife only
- Route pattern (from RESEARCH.md ┬зPattern verified from admin.py):
  ```python
  @router.get("", response_model=list[PatientResponse])
  async def list_patients(params: PatientSearchParams = Depends(), db: AsyncDB, current_user: CurrentUser, _=require_role([...])):
      svc = PatientService(db, current_user)
      return await svc.search(params)
  ```
- `GET /patients/duplicate-check` must be registered BEFORE `GET /patients/{patient_id}` in the router тАФ FastAPI matches routes in order; a literal path segment (`/duplicate-check`) must come before a path parameter (`/{patient_id}`)
- Register router in `backend/app/routers/__init__.py` and include in the main app with `app.include_router(patients_router)` under the `/api` prefix (check how admin router is included in `main.py`)

All CRUD patient API endpoints are functional with RBAC. Duplicate detection returns 409. City-wide search toggle works. `display_id` is computed and returned. Duplicate override writes audit log and sets `possible_duplicate=true`. No BHW access. No lazy-loaded relationships.
