# Plan 03-03: Consultation Backend — Repository + Service + Router

## Objective
Deliver the full consultation backend: `ConsultationRepository`, `ConsultationService` with vitals range validation, and consultation routes nested under `/patients/{patient_id}/consultations` with RBAC.

## Context
- Phase: 03 - Patient ITR + Core Data Model
- Depends on: 03-01 (Consultation ORM model), 03-02 (PatientService and patient schemas already exist — ConsultationService calls PatientRepository to verify patient exists and scoping)
- Blocked by: Nothing beyond 03-01 and 03-02

## Tasks

### Task 1: Create Pydantic schemas for consultations

**Files to create/modify:**
- `backend/app/schemas/patient.py` — add ConsultationCreate, ConsultationResponse (extend the existing schemas file from 03-02)

**Implementation notes:**
- `ConsultationCreate` — request body for POST /patients/{patient_id}/consultations:
  - Required: `chief_complaint: str` — minimum length 1 (validate with `Field(min_length=1)`)
  - All vitals optional with None defaults: `bp_systolic: int | None = None`, `bp_diastolic: int | None = None`, `heart_rate: int | None = None`, `respiratory_rate: int | None = None`, `temperature: Decimal | None = None`, `weight: Decimal | None = None`, `height: Decimal | None = None`
  - `vitals_extra: dict[str, float] | None = None` — for O2 saturation, blood glucose; keys are free-form
  - `diagnosis: str | None = None`
  - `referring_to: str | None = None`
  - Add `@model_validator(mode="after")` for vitals range validation — raise `ValueError` only when a field is provided (not None). Ranges to validate (CONTEXT.md §Vitals Model — "range check only when provided"):
    - `bp_systolic`: 60–250 mmHg
    - `bp_diastolic`: 40–150 mmHg
    - `heart_rate`: 30–250 bpm
    - `respiratory_rate`: 6–60 breaths/min
    - `temperature`: 30.0–45.0 °C
    - `weight`: 0.5–300.0 kg
    - `height`: 30.0–250.0 cm
- `ConsultationResponse` — response for GET and POST consultations:
  - All Consultation model fields: `id`, `patient_id`, `recorded_by_id`, `chief_complaint`, all vitals columns, `vitals_extra`, `diagnosis`, `referring_to`, `created_at`, `updated_at`
  - Computed field `bmi: float | None` — use Pydantic v2 `@computed_field @property`: compute `weight / (height/100)**2` rounded to 1 decimal. Return None if either `weight` or `height` is None or `height` is 0. Do NOT store BMI as a column (CONTEXT.md decision)
  - `recorded_by_name: str | None = None` — full name of the recording user; populated by service via joinedload
  - `model_config = ConfigDict(from_attributes=True)`

### Task 2: Create ConsultationRepository

**Files to create/modify:**
- `backend/app/repositories/patient.py` — add ConsultationRepository class (extend the same file as PatientRepository)

**Implementation notes:**
- `class ConsultationRepository(BaseRepository)` in the same `backend/app/repositories/patient.py` file
- BHS isolation for consultations is enforced indirectly through the parent patient — the service verifies the patient belongs to the user's BHS before allowing consultation creation. `Consultation` has no `health_station_id` column, so `_isolation_filter()` would skip it anyway (the base class checks `hasattr(model, "health_station_id")`)
- **`list_for_patient(patient_id: int, limit: int = 50, offset: int = 0) -> list[Consultation]`**:
  ```python
  stmt = (
      select(Consultation)
      .where(Consultation.patient_id == patient_id)
      .options(joinedload(Consultation.recorded_by))
      .order_by(Consultation.created_at.desc())
      .limit(limit)
      .offset(offset)
  )
  result = await self.session.execute(stmt)
  return list(result.scalars().all())
  ```
  Default sort is newest-first (CONTEXT.md §Patient Profile Page)
- **`create(patient_id: int, recorded_by_id: int, data: dict) -> Consultation`**:
  ```python
  consultation = Consultation(
      patient_id=patient_id,
      recorded_by_id=recorded_by_id,
      **data,
  )
  self.session.add(consultation)
  await self.session.flush()
  await self.session.refresh(consultation)
  return consultation
  ```
  Do not commit — service manages transaction
- **`get_by_id(consultation_id: int) -> Consultation | None`**: For future use (not required for Phase 3 routes but useful for completeness): `select(Consultation).where(Consultation.id == consultation_id).options(joinedload(Consultation.recorded_by))`

### Task 3: Create ConsultationService + consultation routes

**Files to create/modify:**
- `backend/app/services/patient.py` — add ConsultationService class (extend same file as PatientService)
- `backend/app/routers/patients.py` — add consultation sub-routes (extend same file as patient router)

**Implementation notes:**

**ConsultationService (`backend/app/services/patient.py`):**
- `class ConsultationService`: `__init__(self, session: AsyncSession, current_user: UserSchema)` — instantiates `ConsultationRepository` and `PatientRepository` (needs patient repo to verify patient exists and BHS ownership)
- **`async def list_consultations(patient_id: int) -> list[ConsultationResponse]`**:
  1. Verify patient exists: call `PatientRepository.get_by_id(patient_id)` — raise `HTTPException(404)` if None
  2. BHS access check: if user is NOT in CROSS_BHS_ROLES and patient's `health_station_id != current_user.health_station_id`, raise `HTTPException(403, detail="Patient belongs to another BHS")`
  3. Fetch consultations: `ConsultationRepository.list_for_patient(patient_id)`
  4. Map to `ConsultationResponse`, including `recorded_by_name` from `consultation.recorded_by.full_name`
- **`async def create_consultation(patient_id: int, data: ConsultationCreate) -> ConsultationResponse`**:
  1. Verify patient exists (same as above)
  2. BHS mutation guard: if user is NOT in CROSS_BHS_ROLES and patient's `health_station_id != current_user.health_station_id`, raise `HTTPException(403, detail="Cannot add consultation for a patient from another BHS")`. This enforces RESEARCH.md §Pitfall 4 — city-wide search is read-only for BHS-level roles
  3. Note: `recorded_by_id = current_user.id` — use `int(payload["sub"])` already done in `get_current_user()`, so `current_user.id` is the correct INT (RESEARCH.md §Pitfall 6)
  4. `ConsultationRepository.create(patient_id=patient_id, recorded_by_id=current_user.id, data=data.model_dump(exclude={"chief_complaint"}, exclude_none=True) | {"chief_complaint": data.chief_complaint})`
  5. `await self.session.commit()`
  6. Return `ConsultationResponse.model_validate(consultation, from_attributes=True)`

**Consultation routes (added to `backend/app/routers/patients.py`):**
- `GET /patients/{patient_id}/consultations` — roles: nurse, midwife, physician, city_health_officer, disease_surveillance_officer, phis_coordinator
- `POST /patients/{patient_id}/consultations` — roles: nurse, midwife, physician (CONTEXT.md: physician can create consultations)
- Pattern:
  ```python
  @router.get("/{patient_id}/consultations", response_model=list[ConsultationResponse])
  async def list_consultations(
      patient_id: int,
      db: AsyncDB,
      current_user: CurrentUser,
      _=require_role(["nurse", "midwife", "physician", "city_health_officer", "disease_surveillance_officer", "phis_coordinator"]),
  ):
      svc = ConsultationService(db, current_user)
      return await svc.list_consultations(patient_id)

  @router.post("/{patient_id}/consultations", response_model=ConsultationResponse, status_code=201)
  async def create_consultation(
      patient_id: int,
      body: ConsultationCreate,
      db: AsyncDB,
      current_user: CurrentUser,
      _=require_role(["nurse", "midwife", "physician"]),
  ):
      svc = ConsultationService(db, current_user)
      return await svc.create_consultation(patient_id, body)
  ```

## Validation
- [ ] `curl -X POST "http://localhost:8000/api/patients/{id}/consultations" -H "Authorization: Bearer {nurse_token}" -d '{"chief_complaint":"Fever for 3 days","temperature":38.5}'` returns 201 with consultation JSON including computed `bmi: null`
- [ ] `curl -X POST "http://localhost:8000/api/patients/{id}/consultations" -d '{"chief_complaint":"Test","weight":60,"height":165}'` returns 201 with `"bmi": 22.0` (or equivalent computed value)
- [ ] `curl -X POST "http://localhost:8000/api/patients/{id}/consultations" -d '{"chief_complaint":"Test","bp_systolic":300}'` returns 422 (vitals range violation)
- [ ] `curl -X POST "http://localhost:8000/api/patients/{id}/consultations" -d '{"chief_complaint":""}'` returns 422 (min_length=1 validation)
- [ ] `curl -X POST "http://localhost:8000/api/patients/{id}/consultations" -H "Authorization: Bearer {physician_token}"` returns 201 (physician can create)
- [ ] `curl -X POST "http://localhost:8000/api/patients/{id}/consultations" -H "Authorization: Bearer {cho_token}"` returns 403 (CHO read-only)
- [ ] `curl -X GET "http://localhost:8000/api/patients/{id}/consultations" -H "Authorization: Bearer {nurse_token}"` returns 200 with list sorted newest-first

## Definition of Done
Consultation creation and listing endpoints are functional. Vitals range validation rejects out-of-range values when provided but accepts None. BMI is computed in the response. `recorded_by_id` is set from the JWT. Physician can create consultations. CHO/DSO/PHIS can list but not create. BHS-scoped roles cannot add consultations to patients from other health stations.
