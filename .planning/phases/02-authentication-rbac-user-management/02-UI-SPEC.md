## Spacing Scale

Standard 8-point scale. All spacing values are multiples of 4px.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge padding, inline label-to-input gap |
| sm | 8px | Compact button padding, row cell internal padding |
| md | 16px | Default form field spacing, card internal padding |
| lg | 24px | Section gap within a card, modal padding |
| xl | 32px | Card-to-card gap, sidebar section breaks |
| 2xl | 48px | Login card vertical padding |
| 3xl | 64px | Login page centering offset from viewport center |

Exceptions:
- Touch targets for all interactive controls: minimum 44px height on tablet breakpoint (md and above). Form inputs use `h-11` (44px) at tablet breakpoint.
- Role checkbox items in the Create/Edit modal: minimum 40px tap height for field worker tablet use.

## Typography

Four sizes, two weights. All sizes in rem; pixel equivalents at 16px root.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px (0.875rem) | 400 regular | 1.5 | Table cell text, form helper text, description copy |
| Label | 14px (0.875rem) | 600 semibold | 1.2 | Form field labels, table column headers, badge text |
| Heading | 20px (1.25rem) | 600 semibold | 1.2 | Card titles, modal titles, page section headings |
| Display | 28px (1.75rem) | 600 semibold | 1.2 | Login card headline only ("Sign in to LINK") |

Font family declaration in `globals.css`:

```css
@theme {
  --font-sans: "IBM Plex Sans", system-ui, sans-serif;
  --font-mono: "IBM Plex Mono", ui-monospace, monospace;
}
```

Mono font usage: health station ID codes, user ID display in activity log, any UUID or numeric code rendered as data (not UI copy).

## Color

All colors expressed as CSS variables in `globals.css` under `:root` and `.dark`. Never hardcode hex or rgb in component files.

### 60/30/10 Split

| Role | CSS Variable | Light value (OKLCH) | Dark value (OKLCH) | Usage |
|------|-------------|--------------------|--------------------|-------|
| Dominant (60%) | `--background` | `oklch(98% 0.005 240)` | `oklch(13% 0.015 240)` | Page background, login page surround |
| Secondary (30%) | `--card` | `oklch(100% 0 0)` | `oklch(17% 0.015 240)` | Login card, admin panel cards, sidebar, modal backgrounds |
| Accent (10%) | `--primary` | `oklch(42% 0.18 240)` | `oklch(65% 0.16 240)` | Specific elements listed below |
| Destructive | `--destructive` | `oklch(53% 0.22 27)` | `oklch(42% 0.15 27)` | Deactivate button, deactivation confirmation dialog |

Accent (`--primary`) is reserved for these specific elements only:
1. "Sign In" button (login form primary CTA)
2. "Create User" button (user list page action)
3. "Save User" button (modal primary CTA)
4. Active sidebar navigation item indicator (left border or background pill)
5. Focus ring on all interactive controls (`--ring` references `--primary`)

### Project-Specific Semantic Tokens

Defined in `globals.css` as required by CLAUDE.md. Phase 2 only uses `--status-*` tokens (for user active/inactive status badges). Others are defined now for downstream phases.

```css
:root {
  --status-critical: oklch(53% 0.22 27);    /* red — deactivated user, error states */
  --status-warning:  oklch(68% 0.17 68);    /* amber — pending/unverified states */
  --status-safe:     oklch(58% 0.16 145);   /* green — active user, success states */
  --bhs-tier:        oklch(55% 0.14 240);   /* blue — BHS-level scope indicators */
  --cho-tier:        oklch(45% 0.18 270);   /* indigo — CHO-level scope indicators */
}
```

Active user badge: `--status-safe`. Inactive user badge: `--status-critical`. All other status combinations: `--status-warning`.

### Muted and Border Tokens

```css
:root {
  --muted:            oklch(95% 0.008 240);
  --muted-foreground: oklch(48% 0.015 240);
  --border:           oklch(90% 0.01 240);
  --input:            oklch(90% 0.01 240);
  --ring:             oklch(42% 0.18 240);  /* matches --primary */
}
```

## Interaction Contracts

### Login Page

Primary visual anchor: "Sign In" button in `--primary` accent color.

| State | Visual Treatment |
|-------|-----------------|
| Default | Email + password inputs, "Sign In" primary button, full-width |
| Input focused | `--ring` focus ring, 2px offset |
| Input invalid | `border-destructive`, `aria-invalid`, inline error below input |
| Button loading | Button disabled + Spinner (shadcn Spinner component) at inline-start position; label changes to "Signing in..." |
| Auth error (401) | Alert component above form: "Incorrect email or password. Try again." — no hint about which field is wrong |
| Account inactive (401 inactive) | Alert: "Your account is inactive. Contact the system administrator." |
| Success | Full-page redirect — no flash; router push to `/admin/users` (system_admin) or `/dashboard` (all others) |

### User List Table

Primary visual anchor: "Create User" button in `--primary` accent color, top-right of page header.

| State | Visual Treatment |
|-------|-----------------|
| Loading | Skeleton rows — 5 skeleton rows, matching column widths |
| Empty (no users) | `Empty` component: heading "No users yet", body "Create the first user account to get started.", "Create User" button |
| Populated | Sortable columns: Full Name (default sort A→Z), Email, Role(s), BHS Assignment, Status, Created; pagination at 25 rows |
| Row hover | `bg-muted/50` row highlight |
| Inactive user row | `text-muted-foreground` on name and email cells; Status badge uses `--status-critical` |
| Row action menu | DropdownMenu with "Edit" and "Deactivate" (active users) or "Edit" and "Reactivate" (inactive users) |
| Sort active | Column header shows sort icon (Lucide `ArrowUp` / `ArrowDown`) at inline-end |

Columns (exact order): Full Name | Email | Role(s) | BHS Assignment | Status | Created | Actions

Role(s) cell: badges, one per role, using `Badge variant="secondary"`. Maximum 2 badges visible; overflow shown as "+N more" tooltip.

BHS Assignment cell: station name string; "—" (em-dash) for roles that are not BHS-scoped (system_admin, city_health_officer, phis_coordinator when health_station_id is NULL).

### Create/Edit User Modal

Primary visual anchor: "Save User" / "Save Changes" button in `--primary` accent color, modal footer.

| State | Visual Treatment |
|-------|-----------------|
| Open (create) | `Dialog` centered, max-width 540px; title "Create User"; all fields empty |
| Open (edit) | Pre-populated with existing user data; title "Edit User — {Full Name}" |
| system_admin checkbox checked | All other role checkboxes become disabled + `opacity-50`; BHS assignment field hidden with `display:none` |
| BHS field visible (non-admin roles) | Combobox searchable by station name; required when any non-admin role is selected |
| Password field (create) | Required; show/hide toggle (Lucide `Eye`/`EyeOff`) at input-end |
| Password field (edit) | Optional; placeholder "Leave blank to keep current password"; same show/hide toggle |
| Form invalid on submit | Field-level errors below each invalid field; scroll to first error |
| Save loading | "Save User" button disabled + Spinner; other modal controls remain interactive |
| Save success | Modal closes; toast "User created successfully" or "User updated successfully"; user list refreshes |
| Save error (422 system_admin conflict) | Field-level error on role checkboxes: "system_admin cannot be combined with other roles" |
| Save error (409 email duplicate) | Field-level error on email: "This email is already in use" |
| Save error (5xx) | Toast error: "Something went wrong. Try again." |

Modal fields (exact order):
1. Full Name (text input, required)
2. Email (email input, required)
3. Password (password input — required on create, optional on edit)
4. Roles (checkbox group — 7 checkboxes with system_admin exclusivity logic)
5. BHS Assignment (combobox — hidden/disabled when system_admin is selected)

Role checkbox labels (exact strings matching `users.roles` values):
- System Administrator (`system_admin`)
- City Health Officer (`city_health_officer`)
- Physician (`physician`)
- PHIS Coordinator (`phis_coordinator`)
- Disease Surveillance Officer (`disease_surveillance_officer`)
- Nurse / Midwife (`nurse`)
- Barangay Health Worker (`bhw`)

### Deactivation Confirmation Dialog

Use `AlertDialog` (not `Dialog`) — destructive action requires explicit confirmation component.

Dialog title: "Deactivate {Full Name}?"
Dialog body: "This will immediately sign {Full Name} out and revoke their access. Their role: {Role(s)}. This action can be reversed."
Confirm button: "Deactivate User" — `variant="destructive"`
Cancel button: "Keep User" — `variant="outline"`

Reactivation: no confirmation dialog. Reactivate action is non-destructive — execute immediately on click, show toast "User reactivated".

### Activity Log Tab

Primary visual anchor: Timestamp column (leftmost), anchoring the log timeline.

| State | Visual Treatment |
|-------|-----------------|
| Loading | Skeleton rows — 5 rows |
| Empty | `Empty` component: "No activity yet", "User management actions will appear here." |
| Populated | Table: Timestamp | Actor (full name) | Action | Target User | — no pagination initially; load last 100 entries |

Timestamp format: `MMM D, YYYY at h:mm a` (e.g. "Mar 16, 2026 at 2:30 PM") — Philippine timezone (Asia/Manila). No patient PII in log entries.

Action copy examples (exact strings):
- "Created account for {Target Full Name}"
- "Updated account for {Target Full Name}"
- "Deactivated {Target Full Name}"
- "Reactivated {Target Full Name}"
- "Reset password for {Target Full Name}"

## Responsive Contract

Field nurses and staff use tablets (768px–1024px). All surfaces must be tablet-first, not desktop-first.

| Breakpoint | Behavior |
|-----------|----------|
| Mobile (< 768px) | Login: full-screen card, no padding surround. Sidebar: hidden, accessible via hamburger. Table: horizontal scroll within ScrollArea. Modal: bottom sheet (Drawer) instead of centered Dialog. |
| Tablet (768px–1024px) | Login: centered card 440px wide. Sidebar: icon-only collapsed by default, expands on hover/tap. Table: all columns visible with horizontal scroll if needed. Modal: centered Dialog 540px. |
| Desktop (> 1024px) | Login: centered card 440px wide. Sidebar: expanded with labels by default. Table: full columns, no scroll. Modal: centered Dialog 540px. |

Touch targets: all interactive controls (buttons, checkboxes, combobox items) minimum 44px height at tablet and mobile breakpoints.

**Sidebar icon-only accessibility:** When the sidebar collapses to icon-only on tablet/mobile, every nav item icon element must carry `aria-label` matching the nav item's full label text (e.g., `aria-label="Users"`, `aria-label="Activity Log"`). The Lucide icon receives the `aria-label` directly; the surrounding link/button receives `title` with the same string.
